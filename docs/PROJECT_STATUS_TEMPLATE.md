# DiagramAI 项目状态跟踪文档 - 标准模板

> **文档说明**: 本文档是项目状态跟踪的标准模板,用于每个Sprint开始时创建初始状态文档
> **使用方法**: 复制本模板,根据具体Sprint信息填充内容
> **模板版本**: v1.0
> **创建日期**: 2025-10-07
> **维护者**: 项目组

---

## 📋 模板使用指南

### 🎯 何时使用此模板

- ✅ **新Sprint启动时**: 复制模板创建新的PROJECT_STATUS_vX.X.md
- ✅ **版本迭代时**: 基于模板创建新版本的状态跟踪文档
- ✅ **重大里程碑后**: Sprint完成归档后,为下一Sprint创建新文档

### 📝 如何使用此模板

1. **复制模板**: 复制本文档为 `PROJECT_STATUS_v{版本号}.md`
2. **填充项目信息**: 更新项目名称、版本、时间等基本信息
3. **设置Sprint信息**: 填写当前Sprint的目标、任务、时间规划
4. **调整章节内容**: 根据实际需求保留或删除相关章节
5. **更新关键里程碑**: 添加历史成就和当前Sprint目标
6. **设置风险清单**: 识别当前Sprint的潜在风险
7. **建立度量指标**: 设定当前Sprint的质量目标

### 🔑 关键填充点

| 填充项 | 位置 | 说明 |
|--------|------|------|
| **项目版本** | 项目概览 | 如v2.3, v2.4等 |
| **Sprint信息** | 整体进度概况 | Sprint名称、时间、工时 |
| **任务列表** | 当前Sprint任务状态 | 从TASKS文档迁移 |
| **关键里程碑** | 项目概览 | 历史成就+当前目标 |
| **风险清单** | 风险和问题跟踪 | 识别新Sprint风险 |
| **度量目标** | 项目度量指标 | 设定质量目标 |

### 文档结构说明

本模板包含以下核心章节:

1. **项目概览** - 项目基本信息、核心特性、价值主张、关键里程碑
2. **项目总体进度** - 整体进度、Sprint概览、规划时间线
3. **当前Sprint任务状态** - Sprint目标、任务列表、完成情况、开发日志
4. **项目度量指标** - 验收情况、开发效率、质量指标
5. **风险和问题跟踪** - 风险清单、已解决问题、待观察点、问题机制

### 维护建议

**日常更新频率**:
- **任务进度**: 每日更新任务状态和进度
- **开发日志**: 任务完成当天记录执行序列
- **风险清单**: 每周评审和更新风险状态
- **度量指标**: 任务完成后更新验收情况

**重要更新时点**:
- **Sprint启动**: 创建新的状态文档
- **任务完成**: 记录完成明细和开发日志

- **里程碑达成**: 更新关键里程碑列表
